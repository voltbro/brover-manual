# Подключение к устройству

BRover-E5 поддерживает несколько способов удалённого подключения для управления, разработки и передачи файлов. Выбор метода зависит от задач: от выполнения команд до полноценной разработки.

## Доступные методы подключения

1. **SSH (терминал)**
   Базовый способ доступа к системе:

   * выполнение команд
   * запуск ROS-нод
   * диагностика и отладка

2. **VS Code (code-server или Remote-SSH)**
   Среда для разработки:

   * редактирование кода
   * работа с файлами проекта
   * удобная навигация по коду


## Общие требования

Для любого способа подключения необходимо:

* компьютер и ровер должны находиться в **одной сети Wi‑Fi**
* использовать стандартные учётные данные:

  * **Логин:** `pi`
  * **Пароль:** `brobro`
  * **Имя хоста (Hostname):** `broverXX.local` (где `XX` — номер устройства)

## Подключение к точке Wi-Fi по умолчанию

При первом запуске **BRover-E5** пытается подключиться к заранее настроенной сети:

```yaml
SSID: TurtleBro
password: turtlew001
```

или

```yaml
SSID: TurtleBro5G
password: turtlew001
```

:::note
Поэтому мы рекомендуем использовать именно такие настройки Wi-Fi роутера для работы с ровером
:::

## Какой способ выбрать

| Метод      | Когда использовать                           | Сложность |
| ---------- | -------------------------------------------- | --------- |
| **SSH**    | Быстрое подключение, диагностика, управление | 🟢 Легко  |
| **VS Code** | Разработка и работа с кодом                  | 🟡 Средне |


:::tip[Совет]
Для первого подключения используйте SSH, чтобы проверить доступность устройства.
:::
