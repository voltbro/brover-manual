# Подключение по SSH

На **BRover-E5** по умолчанию запущен SSH-сервис, что позволяет подключаться к устройству по сети и работать с ним через терминал.

---

## Подключение по имени хоста (hostname)

Каждому устройству присваивается имя вида:

```bash
broverXX.local
```

где `XX` — уникальный номер устройства.

При корректной настройке сети вы можете подключиться к роверу по имени:

```bash
ssh pi@broverXX.local
```

## Подключение по IP-адресу

### Определение IP-адреса ровера

#### Через веб-интерфейс роутера  
   - Откройте браузер и перейдите в интерфейс управления вашим роутером (обычно `192.168.1.1`, может отличаться).  
   - Авторизуйтесь и найдите раздел **"Список клиентов"** или **"Подключенные устройства"**.  
   - Найдите устройство с именем `broverXX` и запомните его IP-адрес.

На примере Keenetic Hopper:

- Заходим на страницу авторизации и вводим логин и пароль с доступом к веб-интерфейсу

![Страница авторизации роутера Keenetic](../img/keenetic-1.png)

- Переходим в раздел **Мои сети и Wi-Fi -> Список клиентов**

![Раздел списка клиентов Keenetic](../img/keenetic-2.png)

- Смотрим список устройств. Находим ровер и его IP-адрес

![IP-адрес ровера в списке клиентов Keenetic](../img/keenetic-3.png)

## Подключение по IP

В примерах используется IP-адрес `192.168.1.41`.

### Ubuntu

1. Откройте терминал и введите:

```bash
ssh pi@192.168.1.41
```

2. При первом подключении подтвердите ключ:

```bash
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Введите `yes` и нажмите **Enter**.

3. Введите пароль `brobro` и нажмите **Enter**.

4. После подключения вы увидите приглашение вида:

```bash
pi@broverXX:~$
```

### Windows

1. Откройте **Командную строку** или **PowerShell**

2. В открывшемся окне выполните команду:

```bash
ssh pi@192.168.1.41
```

![SSH-connection Windows](../img/connectedSSH-1.png)

3. Введите пароль `brobro`

После успешного подключения появится приглашение терминала ровера.

![SSH-connection Windows1](../img/connectedSSH-2.png)

---

Если подключение не удаётся, проверьте:

* что компьютер и ровер находятся в одной сети
* что ровер включён и полностью загрузился
* корректность IP-адреса или имени устройства
