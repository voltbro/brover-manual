## Настройки сети и SSH

### Подключение к Wi‑Fi

По умолчанию при загрузке Raspberry Pi, установленного в роботе **Brover E5**, система пытается подключиться к Wi‑Fi сети с одним из следующих параметров:

* **SSID:** `TurtleBro`
* **Пароль:** `turtlew001`

или

* **SSID:** `TurtleBro5G`
* **Пароль:** `turtlew001`

Поэтому для первого запуска и быстрой работы с роботом рекомендуется настроить Wi‑Fi роутер именно с такими параметрами. Это позволит подключиться к Brover E5 без дополнительной конфигурации.

---

### Подключение по SSH

Для подключения к роботу по SSH компьютер и Brover E5 должны находиться **в одной локальной сети**.

Откройте терминал на компьютере и выполните команду:

```bash
ssh pi@broverNN.local
```

где `NN` — номер вашего ровера.

**Пароль пользователя:**

```
brobro
```
Чтобы **сменить** локальное имя робота с broverNN на другое, запустите скрипт `~/change_hostname.sh `.

Если подключение по имени хоста не работает, вы можете узнать IP‑адрес робота:

* через веб‑интерфейс роутера;
* с помощью приложения *Angry IP Scanner* или его аналогов.

После этого подключение выполняется напрямую по IP‑адресу, например:

```bash
ssh pi@192.168.1.50
```

> `192.168.1.50` — пример IP‑адреса. В вашей сети он, скорее всего, будет другим.

---

### Настройка подключения робота к другой Wi‑Fi сети

Чтобы изменить Wi‑Fi сеть, к которой подключается Brover E5, необходимо отредактировать файл конфигурации Netplan:

```bash
sudo nano /etc/netplan/20-network-wifi.yaml
```

В конец файла добавьте параметры новой Wi‑Fi сети (SSID и пароль).

:::warning 

* файл `/etc/netplan/20-network-wifi.yaml`, как и любой YAML‑файл, **очень чувствителен к отступам**;
* запрещено использовать символы табуляции (`Tab`);
* для отступов используйте **только пробелы**;
* неправильные отступы приведут к ошибке применения конфигурации сети.
:::

После редактирования файла (пример конфигурации показан ниже на скриншоте) сохраните изменения, закройте файл и перезагрузите робота.
