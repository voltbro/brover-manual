**ROS-топики ровера**

На роботе запущены следующие ROS-топики:
```
/bat
/bhi360/imu
/camera_info
/client_count
/cmd_vel
/connected_clients
/image_raw
/image_raw/compressed
/image_raw/compressedDepth
/image_raw/theora
/image_raw/zstd
/joy
/joy/set_feedback
/m_odom1
/m_odom2
/m_odom3
/m_odom4
/m_odom5
/m_odom6
/m_vel1
/m_vel2
/m_vel3
/m_vel4
/m_vel5
/m_vel6
/odom_pose2d
/parameter_events
/rosout
/user_button
```
Ниже приведено описание основных топиков.

---

**Топики камеры**
```
/image_raw                     тип сообщения sensor_msgs/msg/Image
/image_raw/compressed          тип сообщения sensor_msgs/msg/CompressedImage
/image_raw/compressedDepth
/image_raw/theora
/image_raw/zstd
```
Топики, связанные с работой камеры.
Для большинства задач обработки изображений и передачи видео обычно используются первые два топика:
`/image_raw` и `/image_raw/compressed`.

---

**Топики задания скоростей моторов**
```
/m_vel1
/m_vel2
/m_vel3
/m_vel4
/m_vel5
/m_vel6
```
Топики, в которые публикуются желаемые скорости вращения для каждого мотора.
Эти данные через ноду cyphal_bridge передаются напрямую на драйверы DC-моторов по протоколу Cyphal.

Соответствие моторов:
- /m_vel1 – /m_vel3 — левые колёса (переднее, среднее, заднее)
- /m_vel4 – /m_vel6 — правые колёса (переднее, среднее, заднее)

---

**Топики одометрии моторов**
```
/m_odom1
/m_odom2
/m_odom3
/m_odom4
/m_odom5
/m_odom6
```
Топики, в которые публикуется текущая скорость вращения каждого мотора.
Каждый драйвер мотора передаёт данные по Cyphal, после чего cyphal_bridge преобразует их в соответствующие ROS-топики.

Соответствие моторов:
- /m_odom1 – /m_odom3 — левые колёса (переднее, среднее, заднее)
- /m_odom4 – /m_odom6 — правые колёса (переднее, среднее, заднее)

---

**Одометрия робота**
```
/odom_pose2d    тип сообщения geometry_msgs/msg/Pose2D
```
Топик, в который публикуется текущее положение робота в плоскости:
координаты x, y и угол ориентации θ.

---

**Данные инерциального датчика**
```
/bhi360/imu    тип сообщения sensor_msgs/msg/Imu
```
Топик, содержащий данные с IMU-датчика: ускорения, угловые скорости и ориентацию.

---

**Состояние батареи**
```
/bat    тип сообщения sensor_msgs/msg/BatteryState
```
Топик, в который публикуется информация о состоянии батареи:
напряжение, ток, уровень заряда и другие параметры.

---

**Джойстик**
```
/joy    тип сообщения sensor_msgs/msg/Joy
```
Топик, в который публикуются данные, получаемые с джойстика:
состояние кнопок и осей управления.

---

**Пользовательская кнопка**
```
/user_button
```
Топик состояния пользовательской кнопки:
- true — кнопка нажата
- false — кнопка отпущена


:::info
Для получения подробной информации о любом топике используйте утилиту ROS 2:
```
ros2 topic info <имя_топика>
```
Например:
```
ros2 topic info /odom_pose2d
```
